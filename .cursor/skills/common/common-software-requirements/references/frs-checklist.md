# FRS Checklist

- [ ] Every functional requirement has a unique `SRS-*` ID.
- [ ] Trace chain maps `BRD-OBJ-* -> REQ-* -> AC-* -> SRS-* -> evidence`.
- [ ] Each `SRS-*` is written as a requirement card with statement, priority, status, source, behavior, measurement, and verification lane.
- [ ] Input, validation, output, and error state are explicit.
- [ ] Positive and negative flow paths are described.
- [ ] Complex flows use one actor, one goal, one session.
- [ ] Normal course is separated from alternatives and exceptions.
- [ ] External dependency behavior and timeouts are defined.
- [ ] Permission and role constraints are included.
- [ ] NFRs have numeric threshold and measurement method.
- [ ] Verification lane is mapped for each requirement.
